# System Design Learning Journey

This repo documents my personal journey of learning Low Level Design (LLD) and High Level Design (HLD) from fundamentals to production-level thinking.

I'm not here to memorize patterns. I'm here to understand **why** designs are good or bad, learn tradeoffs, and think like a senior engineer.

---

## What's Inside

### Low Level Design (LLD)

#### OOP Foundations
- Encapsulation, Abstraction, Inheritance, Polymorphism
- Composition vs Inheritance
- Interfaces vs Abstract Classes
- Mini assignments (Social Media, User, Post, Comment)

#### OOP Principles
- Encapsulation — Student, BankAccount, ShoppingCart, PasswordManager
- Abstraction — Animal, Vehicle, PaymentMethod, NotificationSender
- Inheritance — Animal, Employee, Payment hierarchies
- Polymorphism — Animal, Payment, Notification, Report, Storage
- Composition — Car (engine injection), Bird (fly behavior), GameCharacter (strategy injection)

#### SOLID Principles
- **S** — Single Responsibility: StudentManager, OrderService, ATM, ParkingLot, FoodDelivery
- **O** — Open/Closed: Shape calculator, Notification, PricingEngine
- **L** — Liskov Substitution: Bird, PaymentMethod, Rectangle/Square redesign
- **I** — Interface Segregation: Machine, Media, Vehicle
- **D** — Dependency Inversion: Notification, PaymentGateway, Cloud Storage

#### And More:
- Clean Code & Engineering Thinking
- Design Patterns (Factory, Builder, Strategy, Observer, Decorator...)
- Real LLD Problems (Parking Lot, ATM, Movie Booking, Splitwise...)

---

### High Level Design (HLD)

- Core Engineering Foundations (client-server, scalability, CAP theorem...)
- Backend Infrastructure (Redis, Kafka, CDN, API Gateway...)
- Distributed Systems (sharding, replication, consistency...)
- Real System Design (URL Shortener, WhatsApp, YouTube, Uber, Netflix...)
- Interview Mastery
